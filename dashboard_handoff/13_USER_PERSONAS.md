# GOLD360 — User Personas

## Persona 1: Intelligence Analyst

**Role:** Daily monitoring of gold ecosystem
**Goal:** Identify anomalies requiring investigation
**Technical Level:** Medium

### Needs
- Quick overview of current risk landscape
- Drill-down into high-risk mines
- Historical trend analysis
- Export capability for reports

### Dashboard Usage
- **Primary pages:** Overview, Model Performance, Geospatial
- **Frequency:** Daily
- **Session length:** 15-30 minutes
- **Actions:** Filter by province, adjust threshold, export charts

### Key Metrics
- Number of high-risk mines
- Changes in risk distribution
- New anomalies detected

## Persona 2: Policy Maker

**Role:** Designing regulatory interventions
**Goal:** Understand impact of policy changes
**Technical Level:** Low

### Needs
- Scenario simulation (what-if analysis)
- Impact assessment of policy levers
- Clear, non-technical visualizations
- Executive summaries

### Dashboard Usage
- **Primary pages:** Scenario Analysis, Overview
- **Frequency:** Weekly
- **Session length:** 30-60 minutes
- **Actions:** Run scenarios, compare outcomes, generate reports

### Key Metrics
- Scenario outcome projections
- Sensitivity analysis results
- Cost-benefit estimates

## Persona 3: Academic Researcher

**Role:** Publishing methodology paper
**Goal:** Validate model and reproduce results
**Technical Level:** High

### Needs
- Full model transparency
- Reproducible experiments
- Statistical validation
- Methodology documentation

### Dashboard Usage
- **Primary pages:** Model Performance, Explainability, Feature Engineering
- **Frequency:** During research periods
- **Session length:** 1-2 hours
- **Actions:** Run ablation studies, examine SHAP values, validate temporal splits

### Key Metrics
- Cross-validation results
- Ablation study outcomes
- Robustness test results
- Leakage validation checks

## Persona 4: Mining Operations Manager

**Role:** Managing individual mine operations
**Goal:** Benchmark against industry standards
**Technical Level:** Low-Medium

### Needs
- Compare own performance to peers
- Understand risk factors affecting their mine
- Track compliance status
- Identify operational improvements

### Dashboard Usage
- **Primary pages:** Overview, Feature Engineering
- **Frequency:** Monthly
- **Session length:** 15-30 minutes
- **Actions:** Filter by mine, view risk factors, compare metrics

### Key Metrics
- Mine-specific risk score
- Delivery efficiency ranking
- Compliance status

## Persona 5: Compliance Officer

**Role:** Ensuring regulatory compliance
**Goal:** Monitor license status and delivery patterns
**Technical Level:** Medium

### Needs
- Track license status across mines
- Monitor delivery patterns
- Identify compliance violations
- Generate audit reports

### Dashboard Usage
- **Primary pages:** Model Performance, Geospatial
- **Frequency:** Weekly
- **Session length:** 30-60 minutes
- **Actions:** Filter by license status, track delivery gaps, export audit data

### Key Metrics
- Compliance score distribution
- License status breakdown
- Delivery gap trends
